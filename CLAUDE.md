## Architecture

This is a **Nuxt 4 + Prismic CMS** site for Museum Skanderborg, using **UnoCSS** for styling.

### Data flow

Pages fetch content from Prismic via `usePrismic()` and `@prismicio/client`. Each page type (defined in `customtypes/`) maps to a route in `nuxt.config.ts`:

- `page` (uid `home`) → `/`
- `page` (other uids) → `/:uid`
- `event` → `/events/:uid`

Pages render their CMS content through `<SliceZone>`, which maps Prismic slice types to Vue components registered in `app/slices/index.ts`.

### Slice Machine

Slices live in `app/slices/`. Each slice has:

- `index.vue` — the rendered component
- `model.json` — the Prismic field schema
- `mocks.json` — mock data for the Slice Machine simulator

The simulator is accessible at `/slice-simulator`. After editing `model.json` via the Slice Machine UI (port 9999), run the dev command to keep both servers running.

`app/slices/index.ts` is auto-generated by Slice Machine — do not edit it manually.

### Custom types

`customtypes/` holds Prismic document type schemas: `page`, `event`, `event_page`, `staff_member`, `department`, `settings`. `prismicio-types.d.ts` at the root is generated from these and provides TypeScript types.

### Layout & grid system

The layout uses a **12-column CSS grid** defined via CSS custom properties in `app/assets/css/main.css`:

- `--layout-column` — computed column width
- `--layout-gutter` — gap between columns (responsive)
- `--layout-margin` — page-edge margin (responsive)

UnoCSS (`uno.config.ts`) extends these with custom utility classes:

- `w-Ncol`, `px-Ncol`, etc. — size/spacing in column units (1–12). Use this classes when defining the horizontal sizing as much as possible.
- `w-gutter`, `mx-layoutMargin` — keyword-based spacing
- `w-xs`, `p-m`, `gap-l` — named size tokens (`3xs` → `2xl`)
- Breakpoints: `>=656:`, `>=960:`, `>=1280:`, etc. (also `<=Npx:` for max-width) using pixel values from `breakpointList`. Use these for media queries as much as possible.
- Typography shortcuts: `text-heading-0` through `text-heading-5`, `text-xs-400`, `text-s-700`, etc. (fluid clamp-based sizes)

Colors: `accent1`–`accent5` (orange, rose, purple, steel-blue, red), `text`, `lightText`, `whiteText`, `background`.

Fonts: `font-display` = Urbanist (Google Fonts), `font-body` = NueMontreal (local, `app/assets/fonts/`).

### Shared components

- `BaseImage` — wraps `<NuxtImg>` with optional zoom-on-hover and dark overlay; responds to parent `.group:hover`
- `BaseSlider` — horizontal scroll container with prev/next arrow buttons
- `BlockHeading` — section heading with a top rule line, renders Prismic rich text
- `SliderCard` — card used inside `BaseSlider` for event listings
- `BaseLink` — link wrapper

### Dev tools

`GridOverlay` and `PrismicDataInspector` components are rendered client-side only (`<ClientOnly>`) in the default layout when `import.meta.dev` is true.
